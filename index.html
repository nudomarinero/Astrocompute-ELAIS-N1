Astrocompute ELAIS-N1

Information about the SKA-AWS Astrocompute proposal called: "Calibration of LOFAR ELAIS-N1 data in the Amazon cloud". Maintained by J. Sabater (Institute for Astronomy, University of Edinburgh).

LOFAR AMIs

We have created an AMI with the LOFAR software pre-installed: ami-63b89309. It is based on the Ubuntu 14.04 LTS official AMI.

Software installed:

  • casacore libraries.
  • casarest libraries.
  • pyrap.
  • LOFAR software (release 2.15).
  • CASA.
  • LSMTool.
  • LoSoTo.
  • factor.
  • WSClean.
  • Montage.
  • Sagecal
  • Some additional Python libraries like astropy, Jupyter, etc.

There is also an AMI (ami-ccbd96a6) with some more additional software installed that can be used as, for example, a head node. The additional software is:

  • GRID certificates and tools.
  • SRM tools.
  • AWS tools and Python libraries.
  • Postgresql 8.4.10 required by some LOFAR BBS tasks.

Legacy AMIs

We keep some old AMIs for compatibility reasons:

  • LOFAR 2.10 node: ami-c7a3c3a2
  • LOFAR 2.10 head: ami-69dfab0c

Packaging of LOFAR

The radio-astro Ubuntu PPA keeps all the LOFAR dependencies packaged and updated. However, we decided to independently generate the LOFAR package for each release to maintain control of the compilation process. The generation of the packages is explained in: https://github.com/nudomarinero/Astrocompute-ELAIS-N1/tree/master/vagrant
